# Security Profiles

TauriShield uses explicit security profiles to make risk decisions visible in the manifest.

## Strict

Default profile for enterprise applications.

- Requires strict CSP.
- Blocks shell, filesystem, camera, microphone and geolocation.
- Treats clipboard and downloads as findings.
- Designed for apps that handle credentials, customer data or internal dashboards.

## Standard

Balanced profile for internal tools with lower data sensitivity.

- Allows standard CSP.
- Still blocks dangerous native permissions.
- Wildcard subdomains remain discouraged.

## Kiosk

Profile for read-only or wallboard style applications.

- Requires strict CSP.
- Blocks all optional permissions.
- Generated windows are fullscreen by default.
- Intended for dashboards, status pages and SOC wallboards.

## Forbidden by design in v0.3 beta

The following capabilities are not generated by the builder:

- Shell execution
- Filesystem access
- Camera
- Microphone
- Geolocation
- Auto updater
- Telemetry

If a future version needs any of these permissions, it must be introduced behind a separate policy profile and ADR.
